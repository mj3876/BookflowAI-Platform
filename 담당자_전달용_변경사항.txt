8. 현재 대시보드 접근 구조 확인

- 현재 대시보드 URL:
  - https://bookflow.myosoon.store/books
- 현재 이 주소는 퍼블릭 접근 구조입니다.
- 14페이지의 목표 구조는 다음과 다릅니다.
  - 직원 AWS Client VPN 연결
  - Private DNS 접근
  - Internal ALB 또는 Internal Ingress
  - auth-pod
  - Azure Entra ID OIDC
  - JWT 발급 후 dashboard 접근
- 현재 확인된 차이:
  - Client VPN은 구축 및 접속 성공
  - auth-pod / dashboard-svc Ingress 매핑은 파일상 존재
  - 그러나 dashboard 접근 경로는 아직 Private DNS/Internal LB가 아니라 public domain + internet-facing LB 구조
- 따라서 14페이지 구조와 완전히 일치시키려면 추가 작업이 필요합니다.

10. 주의사항

- 직원용 dashboard를 internal로 바꾸면 기존 public 접속 테스트 방식이 바뀔 수 있습니다.
- Publisher 외부 접수용 public ALB는 별도 경로로 유지해야 합니다.
- EKS Ingress를 internal로 바꾸는 작업은 LoadBalancer 재생성이 발생할 수 있으므로 담당자와 영향 범위 확인 후 진행하는 것이 좋습니다.
- Client VPN 인증서와 ovpn 파일은 민감 정보이므로 외부 공유 시 주의가 필요합니다.
