#### 1-1. forecast-trigger Lambda → Step Functions 미실행

**증상**
EventBridge 크론(16:29 KST) 실행 기록은 있으나 Step Functions execution 없음. 에러 3회.

**원인**

```

is not authorized to perform: states:StartExecution on resource: bookflow-etl3
```

**해결**

```json
{
  "Effect": "Allow",
  "Action": "states:StartExecution",
  "Resource": "arn:aws:states:ap-northeast-1:354493396671:stateMachine:bookflow-etl3"
}
```

#### 1-2. rds-inventory-seed Glue Job FAILED (SystemExit: 0)

**증상**
데이터 이미 존재 → 스킵 분기에서 `sys.exit(0)` 호출 → Glue 상태 FAILED

**원인**
Glue는 `sys.exit(0)`도 비정상 종료로 처리함

**해결**
`sys.exit(0)` 제거, if/else 구조로 변경

```python
# Before (FAILED)
if cnt > 0:
    job.commit()
    sys.exit(0)

# After (SUCCEEDED)
if cnt > 0:
    print("[rds_inventory_seed] 데이터 존재 — 시드 스킵")
else:
    # 시드 로직 실행
job.commit()
```

### 트러블슈팅

#### Bug 1 — Ubuntu 24.04에서 awscli apt 설치 불가

**증상**: `before-install.sh` exit 100, `E: Package 'awscli' has no installation candidate`

**원인**: Ubuntu 24.04부터 apt 패키지에서 awscli 제거됨

**수정**:
```bash
# 수정 전 (실패)
apt-get install -y awscli

# 수정 후 (AWS CLI v2 공식 installer)
curl -fsSL "https://awscli.amazonaws.com/awscli-exe-linux-x86_64.zip" -o /tmp/awscliv2.zip
unzip -q /tmp/awscliv2.zip -d /tmp/awscli-install
/tmp/awscli-install/aws/install
```

---

#### Bug 2 — python3-venv 미설치로 venv 생성 실패

**증상**: `after-install.sh`에서 `python3 -m venv` 실패

**원인**: `if ! command -v python3` 조건으로 설치 블록이 스킵됨. Ubuntu 24.04는 python3가 기본 설치되어 있어 조건이 false → python3-venv는 별도 패키지이므로 누락

**수정**: 조건문 제거, 무조건 설치
```bash
# 수정 전
if ! command -v python3 >/dev/null 2>&1; then
  apt-get install -y python3 python3-pip python3-venv
fi

# 수정 후
apt-get install -y --no-install-recommends \
  python3 python3-pip python3-venv python3.12-venv \
  libpq-dev gcc unzip
```

---

#### Bug 3 — Secrets Manager describe-secret IAM 권한 없음

**증상**: dev 기본값으로 fallback, Secrets Manager 시크릿이 있음에도 무시됨

**원인**: 기존 코드에서 `describe-secret`으로 존재 여부를 먼저 확인했으나, EC2 IAM Role에 `secretsmanager:GetSecretValue`만 부여되어 있고 `DescribeSecret`은 없음

**수정**: `describe-secret` 제거, `get-secret-value` 직접 호출 후 결과 비어있으면 fallback
```bash
# 수정 전
if aws secretsmanager describe-secret --secret-id "$SECRET_NAME" 2>/dev/null; then
  SECRET_JSON=$(aws secretsmanager get-secret-value ...)
fi

# 수정 후
SECRET_JSON=$(aws secretsmanager get-secret-value \
  --secret-id "$SECRET_NAME" \
  --query SecretString \
  --output text 2>/dev/null || true)
if [ -n "$SECRET_JSON" ]; then
  ...
fi
```

#### Bug 4 — 502 Bad Gateway (uvicorn이 env 파일 읽기 실패)

**증상**: ALB → EC2 정상, nginx → uvicorn 502

**원인**: `/etc/publisher-api.env`가 `chmod 600` (root 전용)인데, uvicorn은 `www-data` 사용자로 실행되어 파일 읽기 불가

**수정**:
```bash
# 수정 전
chmod 600 "$ENV_FILE"

# 수정 후
chown root:www-data "$ENV_FILE"
chmod 640 "$ENV_FILE"
```

---

#### Bug 5 — migration.sql Python 스크립트 NameError (RDS_HOST)

**증상**: `NameError: name 'RDS_HOST' is not defined`

**원인**: bash heredoc 안에서 Python f-string `f'host={RDS_HOST}'` 사용. bash가 `{}` 없이 `$RDS_HOST`를 치환하지 않아 Python 변수명이 빈 문자열로 치환됨

**수정**: bash 변수 치환 대신 `os.environ`으로 읽도록 변경
```python
# 수정 전
conn = psycopg.connect(f'host={RDS_HOST} ...')

# 수정 후
conn = psycopg.connect(
    host=os.environ['RDS_HOST'],
    port=os.environ.get('RDS_PORT', '5432'),
    dbname=os.environ.get('RDS_DB', 'bookflow'),
    user=os.environ['RDS_USER'],
    password=os.environ['RDS_PASSWORD'],
    sslmode='require',
    connect_timeout=5
)
```

---

#### Bug 6 — 500 Internal Server Error (ON CONFLICT 불가)

**증상**: POST `/api/v1/new-book-requests` → 500

**원인**: FastAPI main.py에서 `ON CONFLICT (isbn13) DO NOTHING` 사용하는데, `new_book_requests.isbn13`에 UNIQUE 제약이 없어 쿼리 실패

**수정**: migration.sql에 UNIQUE 제약 추가 (멱등성 보장)
```sql
DO $$
BEGIN
    IF NOT EXISTS (
        SELECT 1 FROM pg_constraint
        WHERE conrelid = 'new_book_requests'::regclass
          AND contype = 'u'
          AND conname = 'new_book_requests_isbn13_key'
    ) THEN
        ALTER TABLE new_book_requests
            ADD CONSTRAINT new_book_requests_isbn13_key UNIQUE (isbn13);
    END IF;
END$$;
```
